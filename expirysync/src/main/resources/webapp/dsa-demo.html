How Stack Works in ExpirySync:

When you add a product in the frontend, the backend Java code:

  1. Creates a Product object with all details
  2. Pushes it onto the InventoryStack (LIFO structure)
  3. Updates the product status based on expiry date
  4. Recent additions are always at the top of the stack

Real use: The "Recent Products" section uses stack operations.

How Queue Works in ExpirySync:

When processing expired products, the backend:

  1. Adds products to ExpiryQueue based on expiry date
  2. Processes them in FIFO order (oldest expiry first)
  3. Applies discounts based on how expired they are
  4. Removes processed products from the queue

Real use: The "Process Expired" button uses queue operations.

How Priority Queue Works in ExpirySync:

For urgent product handling, the backend:

  1. Calculates priority based on days until expiry
  2. Inserts into min-heap (lower number = higher priority)
  3. Always processes product with highest priority (lowest number)
  4. Maintains heap property after each operation

Real use: The "Most Urgent Product" feature uses priority queue.
